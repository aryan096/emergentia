# v0.3 — Sunlight Field, Vapor & Evaporation

## What was built

### New cell type: Vapor (CellType = 5)
- Density 0.0 (lighter than Air at 0.001) — always rises above air
- `CELL_IS_FLUID = true` — air can pass through vapor
- Color: `[200, 210, 220]` (pale blue-grey)
- Opacity: `0.05` — barely attenuates sunlight

### SunlightField (`simulation/Sunlight.ts`)
- `Float32Array` buffer, one value per cell (0.0–1.0 = fraction of surface light received)
- `getSunIntensity(tickCount)` — sinusoidal day/night cycle over 7200 ticks (~2 min at 60 tps)
  - `Math.max(0, Math.sin(phase * Math.PI * 2))` — positive half = day, negative clamped to 0 = night
- `propagate(grid, tickCount)` — top-down column scan; each cell stores the light it *receives* before its own `opacity` attenuates for cells below
- Single buffer (no double-buffering needed — pure function of grid + tickCount, recomputed each tick)

### VaporRules (`simulation/VaporRules.ts`)
- Processed **top→bottom** (mirrors air bubble pattern)
- Multi-step straight rise: tries up to `VAPOR_RISE_SPEED = 10` cells/tick, farthest first; all intermediate cells must be clear air
- Fallback: diagonal rise one step (L/R order randomised to avoid bias)
- **Condensation** (top 25% of grid only): vapor with ≥3 vapor neighbours has 2% chance/tick to become Cloud; being adjacent to an existing cloud adds +15% boost
- Uses `isAir()` which reads the NEXT buffer, so cascading rise works in one tick
- Vacated cell uses `setUnclaimed()`, destination uses `set()` (claims cell)
- Blocked vapor marks itself with `set()` to prevent double-processing

### Cloud cell type (CellType = 6)
- Density 0.0, `CELL_IS_FLUID = false` — blocks water and air flow
- Color: `[240, 245, 250]` (near white), opacity: `0.05`

### CloudRules (`simulation/CloudRules.ts`)
- Processed after VaporRules each tick so newly condensed clouds don't immediately act
- **Stability**: needs `CLOUD_EVAP_THRESHOLD = 100` cloud neighbours to resist evaporation; under-supported cells have `CLOUD_EVAP_RATE = 0.003` chance/tick to revert to Vapor
- **Active growth** (top 25% only): each cloud cell scans 8 neighbours and absorbs adjacent unclaimed vapor at `CLOUD_ABSORB_RATE = 0.08` (8% per neighbour per tick)
- **Drift**: `CLOUD_DRIFT_RATE = 0.012` chance/tick to shift one cell sideways into air
- `isCloud()` helper added to `RuleUtils.ts`

### Evaporation (in `WaterRules.ts`)
- Added `BASE_EVAP_RATE = 0.005` to `WATER_CONFIG`
- Fires at the "idle" step — only water that is *not* moving this tick can evaporate
- `random() < sunlight.get(x, y) * BASE_EVAP_RATE` — probability scales linearly with received light
- `applyAllWaterRules` and `applyWaterRules` now take `sunlight: SunlightField` as extra param

### Visual rendering (`CanvasRenderer.ts`)
- `render(grid, sunlight?)` — optional sunlight param
- Per-cell brightness: `AMBIENT + (1 - AMBIENT) * sunlight.get(x, y)` where `AMBIENT = 0.15`
- Cells in shadow are dim but never pitch black; full sunlight = full color

## Tick order (Simulation.ts)
1. `sunlightField.propagate(grid, tickCount)` — top-down, pure function
2. `applyAllWaterRules(grid, sunlightField)` — bottom→top, includes evaporation
3. `applyAllAirRules(grid)` — top→bottom
4. `applyAllVaporRules(grid)` — top→bottom; condensation into cloud
5. `applyAllCloudRules(grid)` — drift, absorption, evaporation
6. `grid.swap()`

## Known behaviour
- Surface water evaporates gradually during the day, stops at night
- Vapor rises fast (up to 10 cells/tick) and collects in the top quarter of the grid
- In the cloud zone (top 25%), dense vapor masses condense into cloud cells; adjacent clouds accelerate nucleation
- Cloud masses grow by absorbing surrounding vapor; under-supported edge cells slowly erode back to vapor
- Clouds drift slowly sideways; net shape emerges from growth vs. evaporation balance
- Underground water is dim and does not evaporate (receives no sunlight)
- Day/night cycle is visible as a global brightness pulse over ~2 minutes (~7200 ticks)
